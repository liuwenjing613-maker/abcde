# CCAC MAPS 评估指标分析：Macro-F1 为何失效及新指标方案

> 基于 `docs/experiments/submission_log.md` 中 5 次提交的完整诊断
> 日期：2026-06-01

---

## 一、Macro-F1 的三个致命缺陷

### 缺陷 1：奖励"分布赌博"

提交 #4（DeepResidual γ=2.0 + 先验校准）预测 **378/382（99%）** 为"中度"，4/5 类别 F1=0，却拿到最高 MF1=**0.1805**。

| 类别 | F1 (#4) |
|------|---------|
| 中度（292 样本）| **0.86** |
| 正常/轻度/重度/极重度 | **0.00** |

模型本质上做的不是分类，而是根据泄露的测试分布找出了最大类并全量下注。Macro-F1 并未惩罚这种行为。

### 缺陷 2：OOF → 测试无相关性

| 实验 | OOF MF1 | 公榜 MF1 | 衰减 |
|------|---------|-----------|------|
| ND-11 (KD) | **0.329** | 0.177 | ↓ 46% |
| ND-3 (DeepRes γ=2.0) | **0.264** | 0.181 | ↓ 31% |
| ND-1 (BiGRU) | **0.232** | 0.049 | ↓ 79% |

OOF 数据服从**训练分布**（正常 76.5%），公榜服从**不同分布**（中度 76.4%）。Macro-F1 衡量的是模型对训练分布的拟合程度，而非其对测试分布的泛化能力。

### 缺陷 3：阈值/校准依赖

| 模型 | 无校准 MF1 | 先验校准 MF1 | 倍数 |
|------|------------|--------------|------|
| KD (ND-11) | 0.033 | 0.177 | **5.4×** |
| DeepRes (ND-3) | — | 0.181 | — |

同样的模型概率，改变决策边界后 Macro-F1 提升 5.4 倍。说明 Macro-F1 衡量的是**校准**（分布假设），而非**判别能力**（特征提取）。

### 根本原因

Macro-F1 将两个相互独立的质量维度混为一谈：

```
Macro-F1 = f(判别能力, 校准质量, 分布匹配)
```

当训练分布与测试分布相反（多数类互换），这两个维度产生冲突——判别能力强的模型反而可能在分布错误的测试集上表现更差。

## 二、提出的新评估框架

### 原则

1. **分离关注点**：判别、序数正确性、覆盖率各自独立衡量
2. **阈值无关**：主要指标不依赖决策阈值
3. **分布无关**：不应奖励猜测测试分布
4. **门控机制**：任何类别完全失效的模型直接标记为不合格

### 三维指标

#### 指标 1：`macro_auc`（OvR ROC-AUC，按类别取平均）— 主要

**衡量内容**：模型各等级排序能力，与阈值和分布无关。

**定义**：对各类别计算 One-vs-Rest ROC-AUC，再取平均。

**覆盖 Macro-F1 的盲点**：
- 全预测同一类的模型 → AUC ≈ **0.5**（随机），而非虚假的高 Macro-F1
- 阈值无关 → 单纯衡量特征学习质量
- 分布无关 → OOF AUC 应能更好地预测测试 AUC
- 基线：随机猜测 = 0.5，完美 = 1.0

#### 指标 2：`qwk`（二次加权 Kappa）— 序数

**衡量内容**：预测与真实值的序数一致性。

**定义**：Cohen's Kappa 的二次加权变体。权重矩阵为 $w_{ij} = (i - j)^2$，因此"极重度"误判为"中度"（间隔 2）比误判为"重度"（间隔 1）的惩罚重 4 倍。

**为什么重要**：焦虑等级是**有序**的：
```
正常(0) < 轻度(1) < 中度(2) < 重度(3) < 极重度(4)
```
Macro-F1 将所有误分类视为同等严重。QWK 则区分"轻微偏差"与"严重误判"。

AdoDAS 赛道 A2 已采用 QWK 作为主要指标。

#### 指标 3：`min_class_recall`（最小每类召回率）— 门控

**衡量内容**：是否有任何类别被完全忽略。

**规则**：若任一类别 `recall == 0`，模型为**不合格**——无论 AUC/QWK 如何。

**为什么重要**：提交 #4 的 QWK 可能很低（将大量"极重度"判为"中度"），但 Macro-F1 并未将其标记为完全失效。

### 复合评分：`robust_score`

```
robust_score = macro_auc × coverage_penalty
coverage_penalty = 0.5 if min_class_recall == 0 else 1.0  # 召回缺口折半
```

单纯一个数值用于快速排序；完整评估需查看所有三个维度。

## 三、现有提交对比（推测 → 实测验证）

将框架应用于 5 次 CodaBench 提交（基于 OOF 概率）：

| # | 方法 | 旧 MF1 | AUC | QWK | min_recall | robust_score | 合格？ |
|---|------|--------|-----|------|------------|-------------|-------|
| 4 | DeepRes + 先验 (ND-3) | **0.181** | **0.593** | 0.033 | 0.138 | 0.593 | ✓ |
| 3 | KD + 先验 (ND-11) | 0.177 | **0.745** | 0.077 | 0.019 | 0.745 | ✓ |
| 1 | DeepRes + DASS | 0.091 | 0.728 | 0.189 | 0.111 | 0.728 | ✓ |
| 2 | BiGRU 基线 | 0.049 | 0.587 | 0.062 | 0.111 | 0.587 | ✓ |
| 5 | KD（无校准） | 0.033 | ~0.54 | ~0.02 | 0.000 | ~0.27 | ❌ |

**关键洞察**：提交 #4（Macro-F1 最佳）的 AUC 仅为 **0.593**——仅略优于随机（0.5）。Macro-F1 排名完全颠倒：真正的第一（AUC=0.745）是 ND-11，它在 Macro-F1 排名中仅为第三。

> 注：AUC 值来自 OOF 概率，而非 CodaBench（CodaBench 仅反馈离散预测的逐类指标）。测试 AUC 可能因分布偏移而进一步下降。

### ⚠️ 新指标的已知盲点：特征可用性

提交 #1（DeepRes + DASS）在新指标下得分为 **0.728，标记为 ✓**，但实际公榜 MF1 仅 **0.091**——灾难性失败。这暴露了新指标的一个根本性局限：

**OOF 评估无法检测"特征捷径"问题。** 当模型依赖于训练时可用的特征、但测试时该特征缺失时，任何 OOF 指标都会高估测试表现。

| 失败模式 | 根本原因 | Macro-F1 能检测吗？ | AUC 能检测吗？ |
|----------|----------|---------------------|----------------|
| 分布赌博 | 全量预测少数类 | ❌ 反而奖励 | ✅ AUC ≈ 0.5 |
| 分布偏移 | 训练多数类 ≠ 测试多数类 | ❌ | ⚠️ 部分（需与测试分布解耦） |
| **特征捷径** | **依赖了测试时不可用的特征** | **❌** | **❌** |

**根本原因**：OOF 数据服从训练分布（特征完整），其定义上无法模拟测试时特征的缺失。这不仅限于 AUC——Macro-F1 同样无法解决：该模型的 OOF MF1 为 0.363，在 Macro-F1 排名中位列第一。

**正确方案**：

1. **在评估期间始终禁用不可用的特征。** 所有无 DASS 实验（ND-*）已经做到了这一点——若发现 DASS 在测试中不可用，则在训练时即将其移除。在这些实验中，AUC 实现了有效排序。

2. **按特征集分组。** 比较仅在相同输入特征的实验之间才有意义：

   | 特征集 | OOF MF1 范围 | OOF AUC 范围 | 测试适用？ |
   |--------|-------------|-------------|-----------|
   | AV + DASS | 0.34–0.36 | 0.70–0.75 | ❌ 无效 |
   | 仅 AV（无 DASS） | 0.24–0.31 | 0.59–0.76 | ✅ |

3. **若训练时包含某个特征、但测试时将其置零，则该实验视为无效。** 其指标仅对模型开发有意义，不应与真实的测试条件实验进行比较。

**底线**：AUC 解决了"分布赌博"问题（Macro-F1 完全遗漏），但**并未声称能解决**"特征捷径"问题——这本质上是一个实验设计问题，而非指标问题。正确的做法是仅使用在测试时可用的特征进行训练，然后使用 AUC 对不同方法进行排序。

```python
def macro_auc(probabilities, labels, num_classes) -> float
```

## 五、开放问题

1. **AUC 需要概率值**：CodaBench 仅返回离散预测的逐类指标。我们可以在离线评估时计算 AUC，但无法通过 CodaBench 反馈进行验证。可以通过提交含概率的 zip 包，自行计算测试 AUC 吗？——可能不行。

2. **复合评分权重**：`coverage_penalty = 0.5` 为任意设定。是否应使用更连续化的惩罚，例如 `exp(-k * num_zero_recall_classes)`？

3. **macro_auc 与 QWK 的关联**：若两者高度相关，或许只需保留一个。需在 OOF 概率上进行验证。

4. **OOF 分布 = 训练分布**：OOF AUC 仅能衡量判别质量，无法直接衡量测试集分布偏移下的表现。但这正是我们需要的——将判别与校准解耦。

## 六、结论

| 维度 | Macro-F1（旧） | Robust Score（新） |
|------|---------------|-------------------|
| 阈值依赖 | 是 | 否（AUC） |
| 分布依赖 | 是 | 否 |
| 奖励赌博 | 是 | 否（全猜一类 AUC→0.5） |
| 序数感知 | 否 | 是（QWK） |
| 检测类别崩塌 | 否 | 是（min_recall） |
| 单一数字排序 | 是 | 是（robust_score） |

**核心思想**：将"模型能否区分各个类别？"（AUC）与"模型是否猜对了正确的类别？"（校准/F1）解耦。前者衡量学习到了什么；后者衡量对分布的适配程度。在分布发生偏移的竞赛环境中，这两个问题必须分开回答。
