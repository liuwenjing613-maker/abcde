# CCAC MAPS 基线模型架构详解

## 任务定义

**输入**: 受试者在 T1、T2、T3 三个阶段的音视频特征（每个阶段 4 个片段）
**输出**: 受试者在 T4 阶段的焦虑等级（五分类）

| 类别索引 | 标签 | 含义 |
|---|---|---|
| 0 | 中度 | 中度焦虑 |
| 1 | 正常 | 无焦虑 |
| 2 | 轻度 | 轻度焦虑 |
| 3 | 重度 | 重度焦虑 |
| 4 | 非常严重 | 极度焦虑 |

---

## 数据流

### 输入张量

```
形状: [Batch_Size, 3 stages, 4 clips, 3840 dims]

3840 = 3072 (audio_wavlm_base) + 768 (video_dinov2_small)

时间维度: T1 → T2 → T3（纵向追踪 3 个阶段）
片段维度: A01(朗读), B01(昨日), B02(开心), B03(悲伤)
```

每个阶段 × 片段的特征向量是**池化后的固定维度向量**（`pooled.npy`），代表整段音视频的全局统计量，不包含帧级时间信息。

### 缺失处理

部分受试者在某些阶段/片段可能缺失数据。缺失位置：
- 特征值用 **0 填充**
- 通过 `clip_mask [Batch, 3, 4]` 布尔张量标记

## 模型架构

### 总览

```
输入 [N, 3, 4, 3840]
      │
  ┌───┴───────────────────────────────────────────┐
  │  Clip Encoder（权重共享，处理所有 12 个片段）   │
  │  Linear(3840→256) → LayerNorm → GELU → Dropout │
  └───┬───────────────────────────────────────────┘
      │  [N, 3, 4, 256]
      │
  ┌───┴───────────────────────────────────────────┐
  │  Attention Pooling（在每个阶段内部融合 4 片段） │
  │  Linear(256→1) → Masked Softmax → Weighted Sum │
  └───┬───────────────────────────────────────────┘
      │  [N, 3, 256]  ← 每个阶段一个向量
      │
  ┌───┴───────────────────────────────────────────┐
  │  + Stage Position Embedding（可学习 [3, 256]） │
  └───┬───────────────────────────────────────────┘
      │  [N, 3, 256]
      │
  ┌───┴───────────────────────────────────────────┐
  │  BiGRU（单层, hidden=192, bidirectional=True） │
  └───┬───────────────────────────────────────────┘
      │  [N, 3, 384]  ← 每个时间步 384 维
      │
  ┌───┴───────────────────────────────────────────┐
  │  特征融合                                       │
  │  ├── GRU 输出均值池化      → [N, 384]          │
  │  ├── Stage 表示均值池化    → [N, 256]          │
  │  └── T3 阶段表示（最后步） → [N, 256]          │
  │  拼接                     → [N, 896]           │
  └───┬───────────────────────────────────────────┘
      │  [N, 896]
      │
  ┌───┴───────────────────────────────────────────┐
  │  分类器                                         │
  │  LayerNorm(896) → Linear(896→256) → GELU       │
  │  → Dropout → Linear(256→5)                     │
  └───┬───────────────────────────────────────────┘
      │  [N, 5]  ← logits
      │
    Softmax → 5 类概率分布
```

## 各组件详解

### 1. Clip Encoder（片段编码器）

```python
nn.Sequential(
    nn.Linear(3840, 256),   # 降维: 3840 → 256
    nn.LayerNorm(256),       # 层归一化
    nn.GELU(),               # 激活函数
    nn.Dropout(0.2),         # 正则化
)
```

**设计意图**: 将所有片段的异构特征（不同音频模型 + 不同视频模型）映射到统一的 256 维语义空间。所有 12 个片段（3 阶段 × 4 类型）**共享同一组权重**，迫使模型学习通用片段表示。

- 输入: `[N, 3, 4, 3840]`
- 输出: `[N, 3, 4, 256]`

### 2. Attention Pooling（注意力池化）

```python
logits = Linear(256 → 1)           # 每个片段一个标量分数
logits = mask_fill(-inf)           # 缺失片段置 -inf
weights = softmax(logits, dim=-1)  # 阶段内归一化
weights = weights * mask / sum     # 重归一化（仅有效片段）
stage_repr = Σ(weights × encoded)  # 加权求和
```

**设计意图**: 在每个阶段（T1/T2/T3）内，模型自动学习如何加权融合 A01/B01/B02/B03 四个片段。例如，情绪诱发片段（B02 开心/B03 悲伤）可能比标准化朗读（A01）对焦虑预测更重要。

- 输入: `[N, 3, 4, 256]`
- 输出: `[N, 3, 256]`

**缺失阶段处理**: 若某阶段四个片段全部缺失，则该阶段表示置为零向量。

### 3. Stage Position Embedding（阶段位置编码）

```python
self.stage_position = nn.Parameter(torch.zeros(3, 256))
nn.init.normal_(self.stage_position, mean=0.0, std=0.02)
```

**设计意图**: 为 T1、T2、T3 三个阶段添加可学习的位置标记，使模型能区分不同时间点。类似 Transformer 的位置编码，但这里只有 3 个位置，直接学习即可。

### 4. Temporal Encoder（时序编码器）

```python
nn.GRU(
    input_size=256,
    hidden_size=192,
    num_layers=1,
    batch_first=True,
    bidirectional=True,
)
```

**设计意图**: 建模 T1 → T2 → T3 的纵向轨迹。双向 GRU 可以捕捉焦虑水平的趋势（上升/下降/稳定）和模式（先升后降等）。

- 输入: `[N, 3, 256]`（3 个时间步）
- 输出: `[N, 3, 384]`（双向拼接，192×2=384）

### 5. Fusion（特征融合）

```python
pooled_temporal = GRU输出在时间维度的均值       # [N, 384]
pooled_stage    = 三个阶段表示在时间维度的均值   # [N, 256]
final_stage     = T3 阶段的表示                   # [N, 256]

fused = Concat([pooled_temporal, pooled_stage, final_stage])  # [N, 896]
```

**设计意图**: 三种粒度的信息互补：
- `pooled_temporal` — GRU 编码后的全局时序摘要
- `pooled_stage` — 原始阶段表示的均值（跳跃连接，保留底层特征）
- `final_stage` — 最后一个时间步 T3 的信息（最近的观测最重要）

### 6. Classifier（分类器）

```python
nn.Sequential(
    nn.LayerNorm(896),
    nn.Linear(896, 256),
    nn.GELU(),
    nn.Dropout(0.2),
    nn.Linear(256, 5),
)
```

**设计意图**: 两层 MLP 将融合特征映射到 5 个类别 logits，中间保留非线性变换能力。

## 训练配置

### 损失函数

```python
CrossEntropyLoss(
    weight=class_weights,        # 逆频率类权重
    label_smoothing=0.0          # 默认无标签平滑
)
```

**类权重计算**:
```python
count[c] = 类别 c 的样本数
weight[c] = total / (num_classes × count[c])
weight[c] = weight[c] ^ class_weight_power  # 默认 1.0
```

### 优化器

- AdamW, lr=1e-3, weight_decay=1e-4

### 交叉验证

- 5 折分层 K-Fold（`StratifiedKFold`），在受试者级别划分
- 若某类别样本数 < 2，退化为普通 `KFold`

### 早停

- 监控指标: Macro-F1（验证集）
- Patience: 12 个 epoch 无提升则停止

### 特征预处理

- 对每折训练集拟合 StandardScaler（均值/标准差），应用于训练集和验证集
- 缺失特征填充为 0

## 推理流程

### 验证集评估（OOF）

```
Fold 1: 训练 [80%] → 预测 [20%] ─┐
Fold 2: 训练 [80%] → 预测 [20%]  │
Fold 3: 训练 [80%] → 预测 [20%]  ├──→ 拼接全部 → OOF 预测
Fold 4: 训练 [80%] → 预测 [20%]  │
Fold 5: 训练 [80%] → 预测 [20%] ─┘
```

### 测试集推理

```
测试特征 → 5 折模型分别预测概率 → 平均 → argmax → 最终类别
```

## 关键设计决策

1. **权重共享**: Clip Encoder 在所有 12 个片段时间共享，减少参数量，强制通用表示
2. **片段内池化优先于时序建模**: 先在每个阶段内做注意力池化（4→1），再对三个阶段做 GRU（3 步）。这使得时序模型只需处理 3 个时间步，而非 12 个
3. **跳跃连接**: 融合层同时使用 GRU 输出和原始阶段表示，避免深层网络中的信息丢失
4. **仅用池化特征**: 每个片段只使用 `pooled.npy`（全局统计量），丢弃帧级 `sequence.npz`。这是为了简单，但也丢掉了片段内的时序动态

---

## 已知局限

1. 未使用 DASS 量表历史（T1-T3 焦虑分数）— 最强的先验信息被忽略
2. 仅使用一种音频 + 一种视频特征 — 12 种特征中只用 2 种
3. 无片段内时序建模 — 每个片段的帧级动态完全丢弃
4. 仅预测 T4 — 无辅助任务（如同时预测 T1/T2/T3 焦虑等级）
